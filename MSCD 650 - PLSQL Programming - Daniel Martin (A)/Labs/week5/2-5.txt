Assignment 8-7 Avoiding recompilation errors
****************************************
****************************************
1) use the %type attribute when declaring variables in a procedure or function.  If the
type within a table changes from, for example, a VARCHAR2 to a NUMBER, the referenced variable
would not be changed as %type pulls the type directly from the object.
2) use "select *" when inserting columns into a procedure.  if a column is added or
changes names, it won't need to be referenced as the procedure is simply selecting
everything within the object.

Assignment 8-8 Identifying the types of dependencies
****************************************
****************************************
Direct Dependencies: A procedure has an object directly named within its definition.  If this
named object is modified, the procedure will become invalidated.  It is validated when run
or if manually compiled.
Indirect Dependencies:  A procedure has an object named, and this object has a direct dependency
on another object, which is the indirect object.  When the indirect object is modified,
both the direct dependency object and the object dependant upon that direct dependency both
will become invalidated.  It is validated if run or manually compiled.
Remote Dependencies: A object referenced within a procedure resides within another database.  If
this object is modified, the procedure becomes invalid.  Unlike the other two, this is not
validated until it is run, meaning the procedure will fail the first time it runs.
